Type safety

Holes in the shared wire contract and casts that hide real shape mismatches.

Serializer erases field types via any

high shared shared-serializer-any-holes

DataMap extends Record<number, any>; serialize/deserialize widen with Record<string, any> and @ts-expect-error. Runtime only checks field count, not types or named keys.

Derive types from a typed schema (mapped tuple / zod) and drop the any intersections.

  • packages/shared/src/network/serializer.ts
  • packages/client/src/network/client_receiver.ts
  • packages/server/src/engine/network/packets/manager.ts

LoadObject.data is unknown[], disconnected from GameObjectData

high shared shared-loadobject-opaque-payload

Shared defines typed player/resource tuples, but the packet stores opaque arrays. Client re-implements ad-hoc guards; server builds arrays by convention. Field order drift is invisible to TypeScript.

Make LoadObject a discriminated union on type with typed payloads (or type-specific packets).

Packet IDs, types, Schema fields, and PacketMaps are four hand-synced tables

medium shared shared-packet-map-manual-sync

Each packet is declared as const ID, type alias, Schema.*.fields, and map entry. Missing one of four yields silent holes; schema field names are untyped strings vs type keys.

Define one schema object that owns id + fields + value types; derive consts/maps/types from it.

  • packages/shared/src/packet_definitions.ts

BasicPoint index signature is any

medium shared shared-basicpoint-index-signature

{ x: number; y: number; [key: string]: any } lets arbitrary properties flow through geometry APIs and defeats strict checking across packages.

Use a plain { x: number; y: number } (or branded type); widen explicitly at call sites that need extras.

Bad patterns

Incorrect or fragile approaches that cause real bugs or hidden coupling.

clamp treats 0 as “no bound” via truthiness

high shared shared-clamp-falsy-zero

min ? Math.max(...) : value skips clamping when min === 0. World movement uses clamp(..., 0, 20000), so the lower bound is a no-op and positions can go negative.

Use min != null / max != null (or Number.isFinite) instead of truthiness.
